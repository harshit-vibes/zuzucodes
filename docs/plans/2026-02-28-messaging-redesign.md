# Messaging Redesign — zuzu.codes Landing Page
**Date:** 2026-02-28
**Approved approach:** C + A (FOMO + Outcome spine, Aspiration-led copy voice)

---

## The Problem with Current Messaging

The current landing page is Python-centric. "Python" appears in the hero headline, section title, track name, FAQ answers, and footer tagline. This is a positioning mistake:

- **Python is the vehicle, not the destination.** The learner's goal is to build things and stay relevant — Python is just how we get them there.
- **The AI wave is underexploited.** The current "Shift" section treats AI as a vague threat to argue against. The real opportunity is naming the FOMO moment directly and owning it.
- **The wrong struggling moment.** The current copy assumes learners have tried tools like Cursor or Copilot. The bigger, more motivated paying segment hasn't even started — they're FOMO-driven beginners who feel left behind by the AI era and are ready to invest in a structured path.

---

## Positioning Framework

### The Positioning Statement (internal)

zuzu.codes is the structured coding platform for the AI era — for self-motivated beginners (students and young professionals) who see AI changing everything around them, feel the urgency to learn to code, and want a serious path from zero to actually building things and advancing their careers. Unlike free tutorials (shallow, directionless) or bootcamps (expensive, overwhelming), zuzu.codes gives them a complete learning system that starts from day one.

### The Core Tension

| Left | Right |
|------|-------|
| Watching AI build things | Building things yourself |
| Prompting and hoping | Understanding and directing |
| Free but aimless | Structured and serious |
| FOMO with no action | FOMO as a starting point |

### Positioning: C + A

- **C spine (FOMO + Outcome):** Name the cultural moment. The AI era is rewarding people who can build. Lead with urgency and opportunity.
- **A voice (Aspiration-led):** Empowering, action-oriented. "Stop watching, start building." No guilt. Pure ambition.

---

## Language Guide

### Use
- **coding / programming / building** — the identity
- **builder** — the aspirational persona
- **the AI era** — the cultural moment
- **structured** — the differentiator against free chaos
- **from zero** — inclusion signal for FOMO beginners
- **real projects / real skills / real things** — outcome language

### Avoid
- ~~Python~~ as the hero of any headline or section title (Python is the *how*, not the *what*)
- ~~AI-native~~ (old positioning)
- ~~operations / automation management~~ (B2B / wrong audience)
- ~~fear language~~ ("don't get left behind", "before it's too late")

### Python Rule
Python is still what we teach. It should appear in:
- Course/track names (factual)
- FAQ answers (factual)
- Feature bullets (factual)

It should NOT headline any section or be the first thing a visitor reads.
